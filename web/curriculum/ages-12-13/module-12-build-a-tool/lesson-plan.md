## Key Concepts

### What Makes a Good AI Tool Idea?
Not every problem needs AI. Before adding AI, ask:

1. **Is it a real problem?** Do actual people have this problem, or are you solving something imagined?
2. **Would AI genuinely help?** Would it be faster, cheaper, or better than existing solutions?
3. **Is it doable with current AI?** Can language understanding, image recognition, or content generation actually help?
4. **Is it ethical?** Does it respect people's privacy, avoid harmful biases, treat people fairly?

> "The worst AI products are solutions looking for a problem. Start with the problem, not the technology."

### Types of AI Tools
Based on what current AI is good at, most AI tools fall into a few categories:

| Type | What it does | Examples |
|---|---|---|
| Answering questions | Provides information on demand | Chatbots, Q&A tools |
| Content generation | Creates text, images, code | Writing assistants, image generators |
| Classification | Sorts things into categories | Spam filters, content moderation |
| Personalisation | Adapts to individual users | Recommendation engines |
| Pattern recognition | Finds trends in data | Medical diagnosis tools, fraud detection |
| Automation | Handles repetitive tasks | Email drafters, scheduling assistants |

### The Simple Prototype Rule
You don't need to build the real thing to test if your idea is good. A **prototype** can be:
- A description of the tool and how it would work
- A mockup drawing of what it would look like
- A manual test — pretend to be the AI and see if people find it useful
- A prompt you write that approximates the tool's behaviour

## Try It — Design Sprint

### Phase 1: Problem Finding (10 mins)

Brainstorm real problems — things that are annoying, inefficient, or hard — in these areas. Write 2–3 ideas in each box:

**At school:**
___

**At home:**
___

**In your community or neighbourhood:**
___

**For a group you belong to (a sport, a hobby, a club):**
___

**For someone you know (a younger sibling, a grandparent, someone with a specific need):**
___

Now pick the one problem you find most interesting. Write it in one clear sentence:

**My chosen problem:** ___

Who has this problem? ___
How often? ___
How painful is it? (1 = minor annoyance, 10 = major daily difficulty) ___/10

---

### Phase 2: Solution Design (15 mins)

Design your AI tool to solve this problem.

**My tool's name:** ___

**In one sentence, what does it do?** ___

**What type of AI does it use?** (refer to the Types of AI Tools table above)
___

**How would someone use it? Describe the user experience in 3 steps:**
1. The user opens the tool and ___
2. The AI then ___
3. The user gets ___

**What data would the AI need to work well?** ___

**One potential problem or concern with this tool:** ___
**How would I address it?** ___

---

### Phase 3: Sketch the Prototype (10 mins)

Draw a simple sketch (it can be very rough) of what your tool looks like. Label:
- Where the user inputs something
- Where the AI output appears
- Any buttons or controls
- What the main screen looks like

This is called a **wireframe** — a bare-bones visual plan.

---

### Phase 4: Test Your Idea (10 mins)

Use AI to pressure-test your concept:

Ask: "I want to build an AI tool that [describe your tool in detail]. Can you help me by:
1. Identifying the biggest potential problems with this idea
2. Telling me if AI is really the right solution or if something simpler would work
3. Suggesting one way to make it more useful
4. Identifying one ethical concern I should think about"

Write the most useful feedback you got: ___
One change you'll make to your design: ___

---

### Phase 5: The Pitch (10 mins)

Write a 1-minute pitch for your tool. Use this structure:

**Problem:** "A lot of people struggle with ___"
**Solution:** "My tool [name] solves this by ___"
**How it works:** "The user ___, and the AI ___"
**Why AI:** "This needs AI because ___"
**One concern and how I've addressed it:** "Some people might worry about ___, so I've thought about ___"

Practice saying your pitch out loud (actually say it) — does it make sense in 60 seconds?

---

## The Ethics Check
Before your tool is "ready," run through this checklist:

- [ ] Does my tool collect personal data? If so, do I need it, and how would I protect it?
- [ ] Could my tool produce different quality results for different groups of people? How would I make sure it's fair?
- [ ] Could anyone use my tool in a harmful way I didn't intend? What safeguards would I add?
- [ ] Am I being transparent with users about what the AI is doing?
- [ ] Would I be comfortable if the people who are most affected saw how my tool works?

## Reflect

2. Did the AI's feedback on your idea reveal anything you hadn't thought of?

3. What's the difference between a useful AI tool and a harmful one? After this module, what's your answer?

## Challenge
**Build the Minimum Viable Prompt:**

A "minimum viable product" (MVP) is the simplest version of something that still works. For your tool idea, create an MVP by writing a detailed prompt that simulates what your AI tool would do.

Test it on at least 3 people. For each person, record:
- Did they find it useful? (yes / somewhat / no)
- What confused them?
- What would make it better?

Then answer: Is your idea actually solving a real problem for real people? What would you change after testing?

People I tested: ___, ___, ___
Their feedback: ___
My revised design idea: ___
